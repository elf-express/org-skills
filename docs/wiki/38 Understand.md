## 快速總結



### 來源和目標



- **來源**：`~/.config/skillshare/skills/` — 編輯技能的地方
- **目標**：AI CLI 技能目錄 — 透過符號連結部署技能



### 同步模式



- **合併**（預設）：每個技能單獨符號鏈接，保留本地技能
- **複製**：每個技能單獨複製，保留本地技能
- **符號連結**：整個目錄是一個符號鏈接



### 追蹤回購



- 使用 `--track` 安裝 Git 儲存庫
- 以 `_` 為前綴（例如，`_team-skills/`）
- 透過`skillshare update <name>`更新



### 技能格式



- `SKILL.md` 與 YAML frontmatter
- 必填：`name`字段
- 可選：`description`，自訂元數據



### 專案技能



- 技能範圍僅限於單一儲存庫（`.skillshare/skills/`）
- 透過 git 與團隊分享 — 當 `.skillshare/` 存在時自動偵測
- 同步模式可依目標配置（合併預設值，符號連結可選）



### 組織技能



- 透過追蹤儲存庫在所有項目之間共用（`--track`）
- 安裝一次，使用`skillshare update --all`更新
- 補充專案技能－標準組織、回購情境項目
